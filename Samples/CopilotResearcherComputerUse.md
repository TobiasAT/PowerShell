# Microsoft 365 & Teams and Entra ID Admin Roles

*Sample for my [blog post about Researcher Agent with Computer Use](https://blog-en.topedia.com/?p=59214).* 

## Overview

The Microsoft ecosystem evolves rapidly. To support internal technical teams, a search was conducted across open-access sources (Topedia Blog, Icewolf Blog and Microsoft Learn) for posts from 2025 that relate to Microsoft 365, Microsoft Teams and Microsoft Entra ID (previously Azure AD).  The aim was to identify new features, licensing changes and role-based access control (RBAC) improvements and produce recommendations.

### Additional Teams insights from Icewolf Blog

Icewolf’s posts provide deeper guidance for Teams administrators:

* **Guest access controls:** Use Microsoft Graph PowerShell to disable guest additions globally or per group by setting `AllowToAddGuests` in `group.unified` settings[3](https://blog.icewolf.ch/archive/2025/08/06/controlling-guest-access-to-your-M365-groups-teams/). This helps ensure that only authorised guests are added to Teams and Microsoft 365 groups.  
*Recommendation*: Evaluate guest access policies and enforce them via Graph API scripts. Document group settings and review them regularly.

* **MicrosoftTeams PowerShell module updates (v7.2.0–v7.5.0):** Icewolf summarises new capabilities: v7.2.0 adds an `EffectivePolicyAssignments` attribute to `Get-CsOnlineUser` and fixes account type issues[22](https://blog.icewolf.ch/archive/2025/07/04/microsoftteams-powershell-module-7-2-0-released/); v7.2.1-preview introduces a `Notify` parameter for phone-number assignments and new Personal Attendant policy cmdlets[21](https://blog.icewolf.ch/archive/2025/07/24/microsoftteams-powershell-module-7-2-1-preview-released/); v7.3.1 fixes `New-Team` for EDU teams[25](https://blog.icewolf.ch/archive/2025/08/22/microsoftteams-powershell-module-7-3-1-released/); v7.5.0 adds parameters for picking up held calls, joining active calls, dynamic caller ID, changing town hall resolution, enabling external presenter verification and adding custom messages for recordings[20](https://blog.icewolf.ch/archive/2025/10/11/microsoftteams-powershell-module-7-5-0-released/).  
*Recommendation*: Keep the module current; adopt new cmdlets and parameters to improve call routing, meeting policy and event management. Update internal scripts to use new flags (e.g., `TownhallMaxResolution`, `UseUnifiedDomain`, `Notify`).

* **Direct routing phone number inventory:** Icewolf demonstrates using PowerShell (`New-CsOnlineDirectRoutingTelephoneNumberUploadOrder`, `Get-CsOnlineTelephoneNumberOrder` and `New-CsOnlineTelephoneNumberReleaseOrder`) to manage and assign direct-routing numbers within Teams[1](https://blog.icewolf.ch/archive/2025/02/04/microsoft-teams-direct-routing-phone-number-inventory/).  
*Recommendation*: For organisations using Direct Routing, adopt these cmdlets to maintain phone number inventories and streamline assignments via Teams admin center.

## Recommendations for technical teams

1. **Review Teams features and plan adoption** – Evaluate which Teams Premium features (Town hall Raise Hand and Polls, prevent screen capture, advanced organiser controls) justify licensing. Provide training on new features such as Loop pages, enhanced meeting search and organiser controls[19](https://blog-en.topedia.com/2025/10/town-halls-in-microsoft-teams-now-let-attendees-request-to-speak/)[18](https://blog-en.topedia.com/2025/10/loop-pages-in-teams-channels-simplify-team-collaboration/)[17](https://blog-en.topedia.com/2025/08/microsoft-teams-simplifies-meetings-with-new-organizer-controls/)[16](https://blog-en.topedia.com/2025/08/enhanced-meeting-search-in-teams-for-windows-and-mac/). Establish guidelines on enabling advanced protection and controlling raise-hand and polls settings.

2. **Optimise licensing and leverage promotions** – With the revival of Teams-included Enterprise suites, compare costs of bundled vs. standalone Teams licences[15](https://blog-en.topedia.com/2025/10/welcome-back-microsoft-365-and-office-365-enterprise-suites-with-teams/). Keep abreast of promotions (Entra Suite, Copilot, Windows 365, Purview) to reduce costs[14](https://blog-en.topedia.com/2025/11/licensing-updates-for-microsoft-cloud-products-november-2025/). Adopt the Office 365 E3 Developer (no Teams) licence for test environments[14](https://blog-en.topedia.com/2025/11/licensing-updates-for-microsoft-cloud-products-november-2025/).

3. **Strengthen application governance** – Determine whether your tenant uses Microsoft-managed or custom app consent policies. For default tenants, Microsoft is now blocking numerous Graph permissions for Outlook and Teams data[13](https://blog-en.topedia.com/2025/11/microsoft-managed-default-app-consent-policy-now-blocks-20-additional-permissions/). Implement admin consent workflows and consider custom policies for fine-grained control. Conduct an audit of third-party apps to ensure they still function correctly.

4. **Migrate to Report Suspicious Activity** – Discontinue the older MFA Fraud Alert feature and enable the Report Suspicious Activity capability before March 2025[12](https://blog-en.topedia.com/2025/02/fraud-alert-feature-in-microsoft-entra-will-be-retired-in-march-2025/). Provide user training on how to report suspicious MFA prompts; new integration with Entra ID Protection helps with automated risk remediation (P2).

5. **Embrace granular Entra ID roles** – Reduce reliance on Global Admin rights by assigning new roles introduced in 2025. For backup operations, designate SharePoint/Exchange/M365 Backup Administrators[11](https://blog-en.topedia.com/2025/10/new-microsoft-entra-id-admin-roles-sharepoint-and-exchange-backup-administrator/). For advanced SharePoint control, use the SharePoint Advanced Management Administrator[10](https://blog-en.topedia.com/2025/11/new-microsoft-entra-id-admin-roles-for-sharepoint-and-copilot-agent-administrators/). For AI agent governance, adopt Agent ID and Agent Registry roles[10](https://blog-en.topedia.com/2025/11/new-microsoft-entra-id-admin-roles-for-sharepoint-and-copilot-agent-administrators/). Evaluate the People Administrator, IoT Device Administrator and Teams Reader roles to delegate appropriate tasks[9](https://learn.microsoft.com/en-us/entra/identity/role-based-access-control/whats-new).

6. **Pilot decentralised identity and verifiable credentials** – Consider using Microsoft Entra Verified ID for secure digital credentials. Icewolf’s guides illustrate advanced setup and use of the Verifiable Credentials Admin API[8](https://blog.icewolf.ch/archive/2025/01/05/verified-ID-advanced-setup/)[7](https://blog.icewolf.ch/archive/2025/01/29/entra-verifieable-credentials-admin-api-with-powershell/). Assess business cases (e.g., onboarding, partner verification) and prepare infrastructure (Key Vault, domain, admin roles). Plan budgets for Face Check pilot (approx. US $0.25 per verification)[6](https://blog.icewolf.ch/archive/2025/08/11/entra-verified-ID-enable-facecheck/).

7. **Manage hybrid identity and group authority** – If operating hybrid AD/Entra environments, explore pilot programs to convert Exchange and group source-of-authority from on-prem AD to cloud control. Ensure Entra Connect Sync or Cloud Sync is updated to version 2.5.76+ and note limitations (no nested group SOA conversion, no extension attributes)[5](https://blog.icewolf.ch/archive/2025/09/19/change-source-of-authority-SOA-for-groups-preview/)[4](https://blog.icewolf.ch/archive/2025/09/16/change-source-of-authority-soa-for-exchange-attribtes-preview/). Assign needed roles (e.g., Exchange Administrator or Recipient Administrator) for these tasks[4](https://blog.icewolf.ch/archive/2025/09/16/change-source-of-authority-soa-for-exchange-attribtes-preview/).

8. **Enhance Teams and M365 governance** – Use PowerShell/Graph API to enforce policies:   
   • **Guest access:** Set `AllowToAddGuests` to false at the tenant or group level for sensitive groups[3](https://blog.icewolf.ch/archive/2025/08/06/controlling-guest-access-to-your-M365-groups-teams/).   
   • **Application tagging:** Use `tags` in service principal manifests to document application ownership and purpose[2](https://blog.icewolf.ch/archive/2025/03/29/set-entra-application-tags-with-powershell/).   
   • **Phone numbers:** Use new Teams cmdlets to manage direct routing inventory and assignments[1](https://blog.icewolf.ch/archive/2025/02/04/microsoft-teams-direct-routing-phone-number-inventory/).

By monitoring community blogs (Topedia, Icewolf) and official Microsoft Learn updates, technical teams can stay informed about new capabilities, licensing changes and security best practices in the Microsoft 365 ecosystem. Implementing granular RBAC roles, updating licences, and adopting new Teams features will enhance collaboration while reducing risk and administrative overhead.
